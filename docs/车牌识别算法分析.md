# 轻量级车牌识别实战 — 文本检测与识别算法分析

## 1. 问题定义

车牌识别是典型的**两阶段文本识别**任务：

```text
输入图像 → [文本检测] 定位车牌区域 → [文本识别] 读出字符 → 输出车牌号
```

与传统 OCR 不同，车牌识别面临三个特殊挑战：

| 挑战 | 说明 |
|------|------|
| **小目标** | 监控视角下车牌通常仅 20-60 像素宽, 特征稀疏 |
| **多角度** | 倾斜、旋转、遮挡，文本行不一定是水平 |
| **光照多变** | 过曝、阴影、夜间补光, 对比度剧烈变化 |

## 2. 文本检测方案对比

本项目实现了双层检测架构：

### 2.1 YOLOv11 车牌检测（主方案）

```text
CCPD2019 (121k 张) → YOLOv11n 微调 → plate_best.pt
                                 ↓
模型架构: 2.59M 参数, 6.4 GFLOPs, 端到端回归 bbox
训练结果: mAP50-95 = 0.98, Precision = 0.999, Recall = 1.0
```

**优点**：
- 单阶段端到端，推理速度快（~1ms/张）
- 直接回归四边形 bbox，对倾斜车牌鲁棒
- 轻量级，适合部署

**局限**：
- 训练数据（CCPD2019 近景车牌）与部署场景（监控远景）存在**域偏移**
- 小目标检测需要更大的输入尺寸或 FPN 多尺度特征

### 2.2 PaddleOCR DB 检测（兜底方案）

当 plate_best.pt 因域偏移无法检出时, 降级使用 PaddleOCR 内置的 **DB（Differentiable Binarization）** 文本检测：

```text
DB 算法核心:
  特征图 → 概率图 P + 阈值图 T → 可微二值化 → 文本区域
  关键创新: 将二值化操作嵌入网络, 端到端可导
```

**为什么 DB 适合兜底**：
- 不依赖特定类别训练, 通用文本检测
- 对任意方向文本有效（DB 输出多边形, 非轴对齐矩形）
- 配合 PP-OCRv4 的 SVTR_LCNet 识别网络, 中文字符识别准确率高

### 2.3 方案对比

| 指标 | YOLOv11 车牌检测 | PaddleOCR DB |
|------|-----------------|-------------|
| 检测速度 | ~1ms | ~5ms |
| 近景车牌(>100px) | 优 | 优 |
| 远景车牌(20-60px) | 差(域偏移) | 中 |
| 中文支持 | 需训练 | 原生 |
| 泛化能力 | 依赖训练数据 | 通用 |
| 误检率 | 低 | 中(车身文字干扰) |

## 3. 文本识别方案

### 3.1 PaddleOCR SVTR_LCNet

本项目 OCR 使用 PaddleOCR 2.7 的 PP-OCRv4 识别模型：

```text
网络结构:
  SVTR (Single Visual Transformer) + LCNet (轻量骨干)
  输入: 3×48×320 (归一化到固定高度)
  输出: 字符序列 + 置信度
```

**SVTR 核心思想**：
- 将文本识别视为序列预测问题
- Transformer encoder 捕捉字符间长距离依赖
- 不需要 RNN, 并行推理更快

**为什么选它**：
- 原生支持中英文混合（中国车牌为省份简称+字母数字）
- 轻量级（模型文件 ~11MB），适合边缘部署
- 自动方向分类（`use_angle_cls=True`）处理倾斜车牌

### 3.2 置信度与过滤策略

PaddleOCR 返回每个检测区域的 `(text, confidence)` 对。本项目实现两级过滤：

```python
# 第1级: 车牌格式正则
_is_plate_format(text) → 匹配中文字符+字母数字模式

# 第2级: 置信度阈值
conf >= 0.3 → 保留
```

## 4. 实验数据与分析

### 4.1 CCPD2019 基准测试

| 指标 | 结果 |
|------|------|
| OCR 准确率 | 90% (9/10 采样) |
| 平均置信度 | 0.980 |
| 最低置信度 | 0.696 |

### 4.2 监控视频实测

| 指标 | 结果 |
|------|------|
| 车辆检测 | 253 辆/视频 |
| 车牌检出率 | ~7% (5帧采样) |
| 误检类型 | 车身喷字(已通过格式过滤抑制) |

**低检出率原因分析**：

```text
CCPD2019 训练数据          实际监控视频
   近景 (>200px)      →     远景 (20-60px)
   正面/小角度         →     高位俯拍
   高清 (720×1160)     →     低清 (1280×674整帧)
```

这是典型的**训练-部署域偏移**问题，不是算法缺陷。

### 4.3 改进方向

1. **多尺度检测**：对车辆区域做 2x 上采样后再做车牌检测
2. **域适应微调**：用监控视频抽帧标注少量车牌，在 CCPD 预训练模型上微调
3. **超分辨率预处理**：对车牌候选区域做轻量 SR 后再 OCR
4. **降低 conf 阈值**：配合更严格的文本格式过滤

## 5. 实战心得

### 5.1 文本检测

- **不要依赖单一方案**。YOLO 检测快但泛化差，DB 检测通用但慢，双层架构取长补短
- **近景训练 ≠ 远景可用**。CCPD2019 精确度 0.98, 不代表实际场景 0.98。需要评估训练数据的**场景分布**
- **小目标检测需要专门优化**。增大输入尺寸、FPN 多尺度、或上采样预处理都是有效手段

### 5.2 文本识别

- **置信度阈值是双刃剑**。设太高漏检，设太低误检。本项目的 0.3 是在 CCPD 测试(avg 0.98)和视频实测之间取的平衡点
- **格式先验很关键**。中国车牌有固定格式(省份+字母+5~6位)，利用这个先验可以过滤掉 90% 的车身文字误检
- **PaddleOCR 开箱即用但版本敏感**。2.7 到 3.6 API 全变了, 生产环境需要锁定版本

### 5.3 工程实践

- **OCR 是全链路瓶颈**。检测 1ms + OCR 5ms = 每辆车 6ms, 10 路视频并行需要 60ms/帧
- **横截线计数比想象中复杂**。车辆 bbox 抖动会产生假越线, 必须加最小位移阈值
- **颜色分类用规则比训练好**。HSV 阈值法零成本、可解释、易调试, 比再训一个分类模型更实用
